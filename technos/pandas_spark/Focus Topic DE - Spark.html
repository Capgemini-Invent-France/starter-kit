Intro

Once upon a time, there was a Panda

  • Pandas is a very useful python library for data manipulation with integrated indexing.
  • Pandas was realeased in 2008
  • All data scientists learn with pandas
  • Pandas does in memory analytics

Your typical pandas code

import pandas as pd

routes = pd.read_csv('routes.csv')
routes = routes.dropna()
result = (
	routes[['route_distance','average_air_time']]
	.groupby(['route_distance'])
	.agg({'average_air_time':['mean', 'median','max','min']}))
print(result)

Ok, what is Spark ?

  • Apache Spark is an open-source unified analytics engine for large scale data processing
  • Spark was realeased in 2014
  • Spark is written in Scala and provides API in Scalar, Java, R, Python
  • Spark usually relies on HDFS (Hadoop Distributed File System) for data storage
  • Spark in-memory processing is a revolution compared to MapReduce

Wait a minute, what is HDFS & MapReduce ?

  • HDFS is about distributed storage
    • Data distributed across several nodes
    • Resilisent, data is replicated
    • Most popluar HDFS frameworks:
  • MapReduce is about distributed processing
    • Map is about distributing work to worker nodes
    • Shuffle is about exchanging data betweed nodes
    • Reduce is about getting the results from workers

Back to Spark

Key points about Spark

  • Sparks needs a dedicated infrastructure to run on
    • GCP - Dataproc
    • ASW - Amazon EMR
    • Azure Spark
    • Palantir Foundry data pipeline are a managed Spark cluster
    • Databricks is a managed spark cluster
  • Spark is lazy
    • meaning that it will only compute when a result is needed
  • Spark driver or executors can run out of memory
    • Yes, Spark is still RAM intensive. If not partitionned well and/or if code not optimized, out of memory can happen
  • Spark is less performant than Pandas for small files
    • Communication costs
  • It is way more performant to compute a column median than a column average in Spark
    • Average operation is commutative, median is not
  • Partitionning has a big impact on Spark performances
    • Communication costs (Shuffling)
  • Sorting partitions can improve Spark performances
    • Communication costs (Shuffling)

Just before we finish

  • Spark is usually used for batch computing, but Spark Streaming exists
  • Spark supports incremental data loading
  • Spark ML is the distributed scikit learns, with much less features, but scalable !
  • GraphX is a dedicated graph module in Spark for distributed graph processing
